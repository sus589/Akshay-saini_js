🔹 1. Function Statement / Function Declaration

👉 Dono same hote hain

function greet() {
  console.log("Hello");
}

👉 isko bolte hain:

Function Statement ✔

Function Declaration ✔

✔ Feature:

Hoisted hota hai (upar aa jata hai)

greet(); // ✔ chalega

function greet() {
  console.log("Hello");
}
🔹 2. Function Expression

👉 Function ko variable me store karna

const greet = function() {
  console.log("Hello");
};

greet();
❌ Hoisting nahi hoti
greet(); // ❌ error

const greet = function() {
  console.log("Hello");
};
🔥 Difference: Function Statement vs Expression
Feature	Function Declaration	Function Expression
Hoisting	✔ Yes	❌ No
Use before declare	✔ Yes	❌ No
Syntax	function name() {}	const a = function() {}
🔹 3. Anonymous Function

👉 Function jiska naam nahi hota

const greet = function() {
  console.log("Hello");
};
❌ Error Case
function () {
  console.log("Hi");
}

👉 ❌ Syntax Error (name missing)

🔹 Anonymous Function ka use

👉 kaha use hota hai?

Callbacks

Event listeners

setTimeout

setTimeout(function() {
  console.log("Hello");
}, 1000);
🔹 Advantages

✔ Short code
✔ One-time use
✔ Clean callbacks

🔹 4. Named Function Expression

👉 Function ka naam bhi hota hai AND variable me assign hota hai

const greet = function sayHello() {
  console.log("Hello");
};
⚠ Gotcha (Important)
const greet = function sayHello() {
  console.log(sayHello);
};

greet();      // ✔
sayHello();   // ❌ Error

👉 function ka naam sirf uske andar accessible hai

🔹 5. Parameters vs Arguments
function add(a, b) { // parameters
  return a + b;
}

add(2, 3); // arguments
Parameters	Arguments
Function define karte waqt	Function call karte waqt
a, b	2, 3
🔹 6. First-Class Functions

👉 JS me functions ko value ki tarah treat karte hain

Example:
function greet() {
  return "Hello";
}

function wrapper(fn) {
  console.log(fn());
}

wrapper(greet);
🔥 Iska matlab:

✔ function variable me store ho sakta hai
✔ function pass ho sakta hai
✔ function return ho sakta hai

🔹 7. Arrow Functions (Modern JS) 🔥

👉 Short syntax wala function

const add = (a, b) => a + b;

console.log(add(2, 3));

👉 Output:

5
🔹 Arrow vs Normal Function
Feature	Normal	Arrow
Syntax	long	short
this	dynamic	lexical
return	explicit	implicit
🔹 Example (this difference)
const obj = {
  name: "Susmita",
  normal: function() {
    console.log(this.name);
  },
  arrow: () => {
    console.log(this.name);
  }
};

obj.normal(); // Susmita
obj.arrow();  // undefined
🔹 Kab aur kyu use hota hai?

👉 Real projects me:

Callbacks → anonymous / arrow

Clean code → arrow

Object methods → normal function

🔹 Final Interview One-Line Answer

👉 Agar interviewer puche:

"Functions in JavaScript?"

👉 bolo:

"Functions in JavaScript are first-class citizens, meaning they can be treated like values. They can be declared, assigned, passed, and returned, and can be written using function declarations, expressions, or arrow functions."
