## in hinglish  in javascript Closures kya hai , aye kis liye use hota hai ? kab and q use hota hai mujhe example deke samjhao aur interview k liye coding question v dedo 

Chalo Closures ko ekdum simple + interview level Hinglish me samajhte hain 👇

🔹 Closure kya hota hai?

👉 Closure = function + uska outer scope (memory)

👉 Matlab:

Jab ek function apne outer function ke variables ko yaad rakhta hai, even after outer function execution khatam ho gaya ho

🔹 Simple Example 🔥
function outer() {
  let count = 0;

  function inner() {
    count++;
    console.log(count);
  }

  return inner;
}

const fn = outer();

fn(); // 1
fn(); // 2
fn(); // 3
🔹 Yaha kya ho raha hai?

👉 outer() execute hua
👉 normally count destroy ho jana chahiye ❌
👉 BUT inner() usko yaad rakhta hai ✔

👉 isi ko bolte hain Closure

🔹 Real Meaning (Short)

👉 Closure allow karta hai:

data ko persist (save) karna

private variables banana

🔹 Real Life Example 🧠
function bankAccount() {
  let balance = 1000;

  return function(amount) {
    balance += amount;
    console.log("Balance:", balance);
  };
}

const account = bankAccount();

account(500);  // 1500
account(200);  // 1700

👉 balance bahar se access nahi ho sakta ❌
👉 sirf function ke through access hoga ✔

🔹 Closure ka use kyu hota hai?
✅ 1. Data Privacy (Important)
function counter() {
  let count = 0;

  return {
    increment: () => ++count,
    get: () => count
  };
}

const c = counter();

console.log(c.increment()); // 1
console.log(c.get());       // 1

👉 direct count access nahi kar sakte → private variable

✅ 2. State maintain karne ke liye

👉 jaise:

counters

timers

API caching

✅ 3. Callbacks / Event handlers me
function setup() {
  let msg = "Hello";

  return function() {
    console.log(msg);
  };
}

const clickHandler = setup();
clickHandler(); // Hello
🔹 Kab use hota hai?

👉 Real projects me:

Angular / React me state manage karne me

Event listeners me

API call caching me

Encapsulation (data hide karna)

🔹 Closure kaise kaam karta hai? (Deep 🔥)

👉 Jab function return hota hai:

uska execution context destroy ho jata hai

BUT agar koi inner function usko use kar raha hai
👉 to JS usko memory me rakhta hai (closure scope)

🔹 Interview Coding Questions 🔥
Q1️⃣ Output kya hoga?
function outer() {
  let x = 10;

  return function inner() {
    console.log(x);
  };
}

const fn = outer();
fn();

👉 Answer:

10
Q2️⃣ Output kya hoga?
function counter() {
  let count = 0;

  return function() {
    count++;
    console.log(count);
  };
}

const c1 = counter();
const c2 = counter();

c1();
c1();
c2();

👉 Answer:

1
2
1

👉 kyunki dono ka alag closure hai

Q3️⃣ Tricky 🔥 (Loop + Closure)
for (var i = 1; i <= 3; i++) {
  setTimeout(function() {
    console.log(i);
  }, 1000);
}

👉 Answer:

4
4
4

👉 kyunki:

var ek hi memory share karta hai

✅ Fix karo (Important)
for (let i = 1; i <= 3; i++) {
  setTimeout(function() {
    console.log(i);
  }, 1000);
}

👉 Answer:

1
2
3
Q4️⃣ Closure based question
function multiply(x) {
  return function(y) {
    return x * y;
  };
}

const double = multiply(2);
console.log(double(5));

👉 Answer:

10
🔹 Interview One-Line Answer

👉 Agar interviewer puche:

"What is closure?"

👉 bolo:

"A closure is a function that remembers and can access variables from its outer scope even after the outer function has finished execution."

👉 Closures powerful hai but:

zyada use karoge → memory leak ho sakta hai

unnecessary variables hold karte rehte hain
