Chalo ab Event Bubbling, Event Capturing, aur Event Delegation ko ekdum simple Hinglish me samajhte hain 👇🔥
(ye DOM + JS interviews ka hot topic hai 😎)

🔥 Sabse pehle: Event Flow kya hota hai?

👉 Jab tum kisi element pe click karte ho, event 3 phases me chalta hai:

Capturing Phase (Top → Down)

Target Phase

Bubbling Phase (Down → Up)

🫧 1. Event Bubbling kya hai?

👉 Event niche se upar jata hai (child → parent → grandparent)

📦 Example:
<div id="parent">
  <button id="child">Click Me</button>
</div>
document.getElementById("parent").addEventListener("click", () => {
  console.log("Parent clicked");
});

document.getElementById("child").addEventListener("click", () => {
  console.log("Child clicked");
});

👉 Output (button click pe):

Child clicked
Parent clicked

👉 Pehle child, fir parent → bubbling

✋ Bubbling rokna (important)
document.getElementById("child").addEventListener("click", (e) => {
  e.stopPropagation();
  console.log("Child clicked");
});

👉 Ab parent trigger nahi hoga ❌

🎯 2. Event Capturing kya hai?

👉 Event upar se niche aata hai (parent → child)

👉 Default me capturing off hota hai

✅ Enable capturing:
document.getElementById("parent").addEventListener(
  "click",
  () => {
    console.log("Parent clicked");
  },
  true // capturing ON
);

document.getElementById("child").addEventListener("click", () => {
  console.log("Child clicked");
});

👉 Output:

Parent clicked
Child clicked

👉 Pehle parent → capturing

🔁 Bubbling vs Capturing (Quick)
Feature	Bubbling	Capturing
Direction	Child → Parent	Parent → Child
Default	✅ Yes	❌ No
Use	Common	Rare
🧠 3. Event Delegation kya hai?

👉 Parent pe event laga ke child ko handle karna

👉 Kyun?
👉 Har child pe alag event lagane ki zarurat nahi

📦 Example:
<ul id="list">
  <li>Item 1</li>
  <li>Item 2</li>
  <li>Item 3</li>
</ul>
❌ Wrong way:
document.querySelectorAll("li").forEach(li => {
  li.addEventListener("click", () => {
    console.log(li.innerText);
  });
});
✅ Delegation (Best way):
document.getElementById("list").addEventListener("click", (e) => {
  if (e.target.tagName === "LI") {
    console.log(e.target.innerText);
  }
});

👉 Ek hi event → sab items handle 👍

🎯 Event Delegation kis liye use hota hai?

Dynamic elements handle karne ke liye

Performance improve karne ke liye

Memory bachane ke liye

🤔 Kab use karte hain?

👉 Jab:

List items dynamically add ho rahe ho

Bahut saare elements ho

Repetitive event listeners avoid karna ho

💡 Q use karte hain?

Less code ✅

Better performance ✅

Easy maintenance ✅

🔥 Real Life Example

👉 Tumhare Angular project me:

Betting list / dynamic UI
👉 Delegation use hota hai internally for performance

🧠 Interview Ready Definition

👉 Event Bubbling: Event propagates from child to parent
👉 Event Capturing: Event propagates from parent to child
👉 Event Delegation: Handling child events using a parent listener

💻 Interview Coding Questions
❓ Q1: Bubbling rokna hai
child.addEventListener("click", (e) => {
  // kya likhenge?
});
✅ Answer:
e.stopPropagation();
❓ Q2: Capturing enable kaise karte hain?
✅ Answer:
element.addEventListener("click", handler, true);
❓ Q3: Delegation ka use karke click handle karo
<ul id="menu">
  <li>Home</li>
  <li>About</li>
</ul>
✅ Answer:
document.getElementById("menu").addEventListener("click", (e) => {
  if (e.target.tagName === "LI") {
    console.log(e.target.innerText);
  }
});
🔥 Tricky Interview Question

👉 Output kya hoga?

document.body.addEventListener("click", () => console.log("Body"));
document.getElementById("btn").addEventListener("click", () => console.log("Button"));

👉 Button click pe:

Button
Body

👉 Reason: bubbling

🚀 Final Summary

👉 Bubbling → niche se upar
👉 Capturing → upar se niche
👉 Delegation → parent se child handle
