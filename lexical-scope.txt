🔹 1. Scope kya hota hai?

👉 Scope = kaha tak variable accessible hai

Types of Scope:
✅ (1) Global Scope
let a = 10;

function test() {
  console.log(a);
}

test();

👉 Output:

10

👉 a har jagah access ho raha hai

✅ (2) Function Scope
function test() {
  let b = 20;
  console.log(b);
}

test();
// console.log(b); ❌ error

👉 b sirf function ke andar accessible hai

✅ (3) Block Scope (let, const)
if (true) {
  let c = 30;
  console.log(c);
}

// console.log(c); ❌ error
🔹 2. Lexical Environment kya hota hai?

👉 Lexical = code likhne ka structure (position)

👉 Simple:

"Function jaha define hota hai, wahi uska scope decide hota hai"

Example 🔥
function outer() {
  let x = 10;

  function inner() {
    console.log(x);
  }

  inner();
}

outer();

👉 Output:

10

👉 kyun?

inner ka lexical parent = outer

isliye x access ho raha hai

🔹 3. Scope Chain kya hota hai?

👉 Jab JS variable dhundta hai:

current scope me check karta hai

nahi mila → parent scope

fir parent → fir global

👉 isi process ko bolte hain Scope Chain

Example 🔥
let a = 1;

function outer() {
  let b = 2;

  function inner() {
    let c = 3;
    console.log(a, b, c);
  }

  inner();
}

outer();

👉 Output:

1 2 3

👉 flow:

c → current scope ✔

b → outer scope ✔

a → global scope ✔

🔹 Diagram samjho (simple)
Global → outer() → inner()
   a        b         c

👉 ye chain = scope chain

🔥 Real Life Example (Closure + Scope)
function counter() {
  let count = 0;

  return function() {
    count++;
    console.log(count);
  };
}

const c = counter();
c(); // 1
c(); // 2

👉 yaha closure + lexical scope ka use ho raha hai

function outer() {
  function inner() {
    console.log(a);
  }

  let a = 2;
  inner();
}

outer();

👉 Answer:

2

👉 lexical scope → inner outer ka variable use karega

Q5️⃣ Closure + Scope
function outer() {
  let x = 10;

  return function() {
    console.log(x);
  };
}

const fn = outer();
fn();

👉 Answer:

10
🔹 Interview One-Line Answer

👉 Agar interviewer puche:

"What is scope chain?"

👉 bolo:

"Scope chain is the process of looking up variables from the current scope to its parent scopes until the variable is found."
