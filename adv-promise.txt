Angular me real API call + Promise vs Observable (interview level) Hinglish me samajhte hain 👇

🔥 Angular me API call kaise hoti hai?

👉 Angular me API call ke liye hum use karte hain:

👉 HttpClient (Observable return karta hai)

📌 Step 1: Service create karo
// user.service.ts
import { HttpClient } from '@angular/common/http';
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class UserService {

  constructor(private http: HttpClient) {}

  getUsers() {
    return this.http.get('https://jsonplaceholder.typicode.com/users');
  }
}

👉 Ye Observable return karega

📌 Step 2: Component me use karo
// app.component.ts
import { Component, OnInit } from '@angular/core';
import { UserService } from './user.service';

@Component({
  selector: 'app-root',
  template: `<div *ngFor="let user of users">{{user.name}}</div>`
})
export class AppComponent implements OnInit {

  users: any[] = [];

  constructor(private userService: UserService) {}

  ngOnInit() {
    this.userService.getUsers().subscribe((data: any) => {
      this.users = data;
    });
  }
}
🤔 Ye Promise nahi Observable kyu hai?

👉 Important interview question 🔥

⚔️ Promise vs Observable (Angular)
Feature	Promise	Observable
Values	1 value	Multiple values
Cancel	❌ nahi kar sakte	✅ unsubscribe
Lazy	❌ eager	✅ lazy
Operators	limited	powerful (map, filter, etc.)
Retry	❌	✅ possible
📦 Promise Example (Angular me)
getUsers() {
  return this.http.get('https://jsonplaceholder.typicode.com/users').toPromise();
}

👉 Use:

async ngOnInit() {
  const data = await this.userService.getUsers();
  console.log(data);
}

⚠️ Note: .toPromise() deprecated hai (interview me bolna 🔥)

🔥 Observable Example (Best Practice)
this.userService.getUsers().subscribe({
  next: (data) => {
    console.log(data);
  },
  error: (err) => {
    console.log(err);
  }
});
💡 Real Use Case (Important)

👉 Tum jo real-time betting app bana rahe ho (socket + live data):

👉 Waha Observable best hai because:

Live updates aate rehte hain

Multiple values stream hoti hai

Real-time UI update hota hai

👉 Promise yaha fail ho jayega ❌

🧠 Interview Ready Answer

👉 "Angular uses Observables instead of Promises because Observables can handle multiple values over time, support cancellation, and provide powerful operators for data transformation."

💻 Interview Coding Questions
❓ Q1: Observable create karo jo 1,2,3 emit kare
✅ Answer:
import { Observable } from 'rxjs';

const obs = new Observable(observer => {
  observer.next(1);
  observer.next(2);
  observer.next(3);
  observer.complete();
});

obs.subscribe(data => console.log(data));
❓ Q2: API data ko filter karo (sirf id > 5)
✅ Answer:
import { map } from 'rxjs/operators';

this.userService.getUsers()
  .pipe(
    map((users: any[]) => users.filter(user => user.id > 5))
  )
  .subscribe(data => console.log(data));
❓ Q3: Promise ko Observable me convert karo
✅ Answer:
import { from } from 'rxjs';

const promise = fetch('https://api.com/data');

from(promise).subscribe(res => console.log(res));
