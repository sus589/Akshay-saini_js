Jab bhi JavaScript code run hota hai, engine usko directly line-by-line execute nahi karta. Sabse pehle Execution Context create hota hai. Isme 2 main phases hote hain:

1️⃣ Memory Creation Phase (Hoisting phase)
2️⃣ Code Execution Phase

Chalo simple example se samjhte hain.

1️⃣ Memory Creation Phase (Hoisting Phase)

Is phase me JavaScript engine variables aur functions ke liye memory allocate karta hai, lekin code run nahi karta.

Rules:

var variables → undefined assign hota hai

function declarations → poora function memory me store ho jata hai

let / const → memory me place reserve hota hai but initialize nahi hota (Temporal Dead Zone)

Example
var a = 10;
var b = 20;

function add() {
  console.log(a + b);
}

add();
Memory Creation Phase me kya hoga
Name	Memory Value
a	undefined
b	undefined
add	function code

Is stage me sirf memory allocate hoti hai.

2️⃣ Code Execution Phase

Ab JavaScript line-by-line code run karta hai.

Step by step
var a = 10;

➡ memory me a = undefined tha
➡ ab 10 assign ho gaya

a = 10
var b = 20;
b = 20
add();

➡ function call hua
➡ new Function Execution Context create hoga

Function ke andar:

console.log(a + b);

Output:

30
Visual Flow
Step 1 → Memory Phase
Global Execution Context

a → undefined
b → undefined
add → function(){...}
Step 2 → Execution Phase
a → 10
b → 20
add() call

Output

30
Ek aur example (Hoisting samajhne ke liye)
console.log(x);
var x = 5;

Memory Phase:

x → undefined

Execution Phase:

console.log(x) → undefined
x = 5

Output

undefined
