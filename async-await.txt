🔥 Async / Await kya hai?

👉 async/await JavaScript ka syntax hai jo Promises ko easy aur readable banata hai

👉 Simple line me:
"Async/await se hum async code ko sync jaisa likh sakte hain"

📌 Basic Rules

async → function ko async banata hai

await → Promise ke result ka wait karta hai

📦 Basic Example
async function test() {
  return "Hello";
}

test().then(res => console.log(res));

👉 Output:

Hello

👉 Note: async function hamesha Promise return karta hai

⏳ Await Example
function getData() {
  return new Promise(resolve => {
    setTimeout(() => {
      resolve("Data aa gaya 🚀");
    }, 2000);
  });
}

async function fetchData() {
  const result = await getData();
  console.log(result);
}

fetchData();

👉 Yaha await wait karega jab tak Promise resolve nahi hota

😵 Without Async/Await (Promise chaining)
getData()
  .then(res => console.log(res))
  .catch(err => console.log(err));
✅ With Async/Await (clean code)
async function fetchData() {
  try {
    const res = await getData();
    console.log(res);
  } catch (err) {
    console.log(err);
  }
}

👉 Clean + readable 👍

🎯 Async/Await kis liye use hota hai?

👉 Async operations ko handle karne ke liye:

API calls 🌐

Database calls 🗄️

File read 📂

setTimeout ⏱️

🤔 Kab use karte hain?

👉 Jab:

Multiple async steps ho

Sequential execution chahiye

Code readable banana ho

💡 Q use karte hain?

Callback hell avoid karta hai ✅

Promise chaining reduce karta hai ✅

Code simple lagta hai ✅

Debugging easy hoti hai ✅

🔥 Real Life Example (Step by Step)
function login() {
  return Promise.resolve("Login success");
}

function getProfile() {
  return Promise.resolve("Profile data");
}

function getPosts() {
  return Promise.resolve("Posts data");
}

async function appFlow() {
  try {
    const loginRes = await login();
    console.log(loginRes);

    const profile = await getProfile();
    console.log(profile);

    const posts = await getPosts();
    console.log(posts);

  } catch (err) {
    console.log(err);
  }
}

appFlow();
⚡ Parallel Execution (important 🔥)

👉 Agar tum chahte ho sab ek saath chale:

async function runParallel() {
  const p1 = getData();
  const p2 = getData();

  const res1 = await p1;
  const res2 = await p2;

  console.log(res1, res2);
}

👉 Best way:

async function runParallel() {
  const results = await Promise.all([getData(), getData()]);
  console.log(results);
}
🧠 Interview Ready Definition

👉 Async/await is a syntactic sugar over Promises that allows writing asynchronous code in a synchronous manner,
 improving readability and maintainability.

⚠️ Important Points (interview me puchte hain)

await sirf async function ke andar use hota hai

async function always Promise return karta hai

Error handle → try...catch

Parallel execution → Promise.all()

💻 Interview Coding Questions
❓ Q1: 2 sec delay ke baad data print karo (async/await)
✅ Answer:
function delay() {
  return new Promise(resolve => {
    setTimeout(() => resolve("Done after 2 sec"), 2000);
  });
}

async function run() {
  const res = await delay();
  console.log(res);
}

run();
❓ Q2: Sequential vs Parallel execution

👉 Sequential:

async function seq() {
  const a = await delay();
  const b = await delay();
  console.log(a, b);
}

👉 Parallel:

async function parallel() {
  const [a, b] = await Promise.all([delay(), delay()]);
  console.log(a, b);
}

async function test() {

  console.log("B");

console.log("C");
test();
console.log("D");

👉 ✅ Output:

A
D
B
🔥 Final Summary

👉 async → function ko Promise banata hai
👉 await → Promise ka result wait karta hai
👉 try/catch → error handling
👉 Promise.all() → parallel execution
